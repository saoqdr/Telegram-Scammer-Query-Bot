# Telegram诈骗犯查询机器人

一个基于 **Telethon + pyTelegramBotAPI + Flask** 的 Telegram 反诈查询机器人与网页查询面板，支持 **机器人查询、网页查询、举报投稿、聊天记录查看、排行榜、统计面板、频道监控、管理后台** 等功能。

## 项目简介

这个项目主要用于做 **Telegram 风险账号 / 诈骗线索查询与展示**。

它包含两部分：

1. **Telegram 机器人端**
   - 用户可以在机器人里直接查询 Telegram ID、用户名、转发消息对象等信息
   - 支持菜单交互、命令交互、按钮回调、举报投稿、赞助入口等

2. **Web 网页端**
   - 提供可视化查询页面
   - 提供排行榜、统计页、聊天记录页、验证页、管理页等功能
   - 适合配合域名、反向代理、1Panel 或 Docker 部署

## 主要功能

### 1. Telegram 机器人查询

- 支持通过 **Telegram ID** 查询
- 支持通过 **@用户名** 查询
- 支持通过 **转发消息** 查询目标账号
- 支持私聊内直接输入内容快速查询
- 支持按钮菜单进入查询、统计、投稿、排行榜、赞助等模块

### 2. 风险信息展示

- 展示目标账号的历史风险记录
- 展示举报 / 投稿结果
- 展示命中情况与关联信息
- 支持查询结果页面化展示

### 3. 投稿 / 举报系统

- 支持用户通过机器人提交投稿 / 举报内容
- 支持管理员审核投稿
- 支持通过回调按钮进行通过 / 拒绝处理
- 可用于维护诈骗账号、风险账号、曝光记录等数据

### 4. 网页查询面板

Web 端提供了多个页面：

- `/`：首页 / 查询入口
- `/query`：风险查询页面
- `/stats`：统计信息页面
- `/leaderboard`：赞助排行榜页面
- `/chatlog`：聊天记录查询页面
- `/verify`：网页验证页面
- `/admin`：后台管理页面

### 5. 聊天记录 / 查询跳转

- 支持基于 token 的聊天记录跳转
- 支持网页端查看指定用户聊天记录
- 支持机器人与网页联动查询
- 适合做风险记录留档和辅助核验

### 6. 统计与排行榜

- 支持查询机器人运行统计
- 支持赞助排行榜展示
- 支持基础数据看板与页面展示

### 7. 赞助 / 支付回调

- 集成了 **OKPay 回调接口**
- 支持赞助功能入口
- 支持排行榜联动展示

### 8. 频道 / 群组监控

- 基于 **Telethon** 进行频道 / 群组数据抓取
- 支持监控指定频道或群组
- 支持管理员添加 / 删除监控目标
- 可用于同步风险数据、抓取曝光内容、辅助查询命中

### 9. 管理功能

项目包含一定的后台管理能力，例如：

- 管理员命令
- 频道增删管理
- 投稿删除 / 审核
- 广播与投票广播
- 聊天记录功能开关
- 抓取功能开关
- 数据库维护命令

### 10. 部署方式

项目支持多种部署方式：

- 直接 Python 启动
- Docker / Docker Compose
- 1Panel 部署
- 域名反向代理访问

## 适用场景

这个项目适合用于：

- 诈骗犯信息查询机器人
- 风险账号曝光平台
- Telegram 反诈辅助查询系统
- 举报 / 投稿 / 审核一体化系统
- 带网页展示的 Telegram 数据查询项目

## 开源说明

这个仓库是 **脱敏后的可公开版本**：

- 已移除真实 Token、API Hash、会话文件、数据库、日志和运行时数据
- 所有敏感配置都应通过 `.env` 注入
- 你需要先复制 `.env.example` 为 `.env`，再填入你自己的配置

## 快速开始

```bash
cp .env.example .env
pip install -r requirements.txt
python zpf_web.py
```

或使用 Docker / 1Panel：

```bash
docker compose up -d
```

## 重要安全提醒

请不要把以下文件提交到 Git：

- `.env`
- `*.session`
- `*.db` / `*.sqlite*`
- `reports.json` / `channels.json`
- 日志文件

## 目录说明

- `zpf_web.py`：机器人主程序（含 Web / 回调 / 查询 / 审核 / 监控逻辑）
- `app.py`：网页查询面板与管理端
- `templates/`：网页模板
- `static/`：静态资源
- `docker-compose.yml`：容器部署示例
- `run.sh`：容器启动脚本
